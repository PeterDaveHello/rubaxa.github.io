JSSDK. Введение.

JSSDK
Введение

Идеология

  • Строгий и единый code-style (JSHint)
  • Тесты до кода (QUnit)
  • Контроль покрытия кода тестеми (Istanbul)
  • В master можно закомитеть только валидный код со 100% покрытием тестами
  • Непрерывная интеграция (Travis-CI)

Начало работы

  1. Клонирование репозитория
  2. npm install — установка зависимостей
  3. grunt init — установка git-хуков и прочего
  4. grunt dev-server — запуск сервера для разработки
  5. grunt — тестирование и сборка проекта

Автоматизация

Если какую-то операцию вы делаете больше двух раз, сделайте таск.

grunt module:create:Model Running "module:create:Model" (module) task Create module Model..OK /git/JSSDK/Model/Model.js..OK /git/JSSDK/Model/Model.test.js..OK /git/JSSDK/Model/README.md..OK

Что уже есть?

Уже есть:

  • config — работа с конфигами/настройками
  • Emitter — «Излучатель»
  • Promise — «Обещания» (Native/jQuery—style)
  • request — Low-level интерфейс для работы с ajax/jsonp запросами
